# AI塔罗牌占卜平台 - 项目规格说明书

## 1. 项目概述

### 项目名称
**AI塔罗占卜平台**（TarotQA）

### 项目定位
纯前端静态Web应用，提供塔罗牌占卜服务。用户可直接在浏览器中使用，无需安装任何后端服务。

### 支持平台
- **Web浏览器**: 面向Windows/macOS/Linux用户，支持Chrome、Firefox、Safari、Edge
- **移动端浏览器**: 面向iOS/Android用户，支持Safari Mobile、Chrome Mobile、Firefox Mobile
- **响应式设计**: 自动检测设备类型，适配桌面/平板/手机三种屏幕尺寸

### 核心价值
- 真实塔罗牌图片 + 专业牌义解读
- 丰富的交互动画效果
- 极低门槛，即开即用
- 单页面应用，无需服务器

## 3. 功能清单

### 3.2 牌阵功能

| 牌阵名称 | 牌数 | 说明 |
|----------|------|------|
| 单牌阵 | 1 | 快速简单占卜 |
| 三牌阵 | 3 | 过去/现在/未来 |
| 凯尔特十字牌阵 | 10 | 深度详细分析 |
| 爱情金字塔 | 4 | 情感专项 |
| 马蹄铁牌阵 | 7 | 运势综合分析 |

### 3.3 占卜流程

```
用户输入问题 → 选择牌阵 → 洗牌 → 抽牌 → 翻牌 → 牌义展示
```

1. **问题输入**：用户描述想要咨询的问题
2. **牌阵选择**：从上述5种牌阵中选择
3. **洗牌动画**：卡牌洗牌动画效果
4. **抽牌**：点击卡牌抽取
5. **翻牌动画**：卡牌翻转展示正/背面
6. **牌义展示**：显示抽中牌的含义

### 3.4 正位与逆位

每张塔罗牌都有**正位**和**逆位**两种状态：

| 状态 | 出现概率 | 含义特点 |
|------|----------|----------|
| 正位 | 50% | 牌面朝上，含义积极、正面 |
| 逆位 | 50% | 牌面旋转180度，含义复杂、挑战 |

**显示规范**：
- **正位**：图片朝上，文字正常显示
- **逆位**：图片旋转180度，文字保持正常方向（在图片下方，不反转）

**数据规范**：
- 每张牌包含 `description`（正位描述）和 `reversedDescription`（逆位描述）
- 逆位标签显示为红色背景，正位标签显示为绿色背景

### 3.5 每日星座运势

- 覆盖十二星座
- 本地运势数据
- 包含：综合运势、爱情运势、事业运势、财运

### 3.6 AI 深度解读功能

#### 3.6.1 功能描述

在占卜结果页面，用户可点击"AI深度解读"按钮获取 AI 生成的深入分析。

**使用流程**：
1. 用户可使用默认 API Key（部署时通过 GitHub Secrets 配置），无需自行配置
2. 也可在"我的"页面配置自己的 MiniMax API Key（优先级更高）
3. 抽取卡牌后，在结果页面点击"AI深度解读"
4. 系统调用 MiniMax API 获取解读内容
5. 解读结果显示在结果下方

#### 3.6.2 API 调用规范

**端点**: `POST https://api.minimaxi.com/v1/text/chatcompletion_v2`

**请求头**:
```
Authorization: Bearer <API_KEY>
Content-Type: application/json
```

**请求体**:
```json
{
  "model": "MiniMax-M2.7",
  "messages": [
    { "role": "system", "content": "<系统提示词>" },
    { "role": "user", "content": "<用户问题+抽卡信息>" }
  ],
  "stream": false,
  "temperature": 1,
  "top_p": 0.95,
  "max_completion_tokens": 2048
}
```

**响应格式**:
```json
{
  "choices": [{
    "finish_reason": "stop",
    "messages": [
      { "role": "assistant", "content": "<AI回复内容>" }
    ]
  }]
}
```

#### 3.6.2 角色系统

AI 塔罗解读采用角色设定，根据不同场景自动选择合适的顾问角色。

**角色列表**：

| 角色 | 适用场景 | 关键词 |
|------|----------|--------|
| 综合顾问 | 单牌阵、三牌阵、通用问题 | 通用 |
| 职涯发展顾问 | 事业抉择、工作问题 | 工作、事业、跳槽、面试、职场 |
| 爱情情感顾问 | 感情占卜、恋爱问题 | 爱情、感情、恋人、复合、桃花 |
| 财富财务顾问 | 财运占卜、投资理财 | 金钱、财富、投资、理财 |
| 决策指引顾问 | 凯尔特十字、重大决策 | 选择、决定、纠结、两难 |
| 综合运势顾问 | 马蹄铁牌阵、综合运势 | 运势、运气、未来、趋势 |

**角色选择规则**：
1. 优先根据用户问题中的关键词匹配
2. 其次根据牌阵类型自动推荐
3. 无匹配时使用默认"综合顾问"

**角色数据文件**：`resources/tarot-personas.md`

#### 3.6.3 错误处理与调试

**错误场景**：

| 错误场景 | 错误信息 | 处理建议 |
|----------|----------|----------|
| API Key 未配置 | 请先在设置中配置 MiniMax API Key | 引导用户去设置页面 |
| API Key 格式无效 | API Key 格式无效，请检查设置 | 提示格式要求 |
| 网络连接失败 | 网络连接失败，请检查网络后重试 | 检查网络 |
| HTTP 401 | API Key 无效或已过期，请检查设置 | 重新配置 Key |
| HTTP 403 | API Key 权限不足 | 检查 Key 权限 |
| HTTP 429 | 请求过于频繁，请稍后重试 | 限流等待 |
| HTTP 500+ | MiniMax 服务器繁忙，请稍后重试 | 服务端问题 |
| 响应格式错误 | 服务器响应格式错误，请稍后重试 | 重试 |
| AI 返回空内容 | AI 暂时无法提供解读，请稍后重试 | 重试 |
| 速率限制冷却中 | 请等待 {n} 秒后再试 | 显示倒计时 |

#### 3.6.6 速率限制

**限制规则**：
- 每 60 秒只能发起一次 AI 深度解读请求
- 限制存储在 localStorage (`ai_last_request_time`)
- 页面刷新后继续有效（跨会话持久化）

**UI 表现**：
- 点击"AI深度解读"按钮后，按钮立即变为灰色禁用状态
- 按钮文字变为 `请等待 {n}s`（倒计时在按钮上实时更新）
- 页面显示浮动倒计时提示作为辅助提醒，格式：`⏳ 请等待 {n}s 后再试`
- 倒计时完成后，按钮自动恢复可用状态，文字变回"AI深度解读"

**倒计时提示样式**：
- 浮动在屏幕中央
- 半透明深色背景
- 金色文字
- 3秒后自动消失

**调试面板**：
- 显示"调试信息"复选框，勾选后展开调试面板
- 调试信息包含：
  - 请求时间戳
  - 牌阵类型
  - 卡牌数量
  - 用户问题
  - 完整请求数据（JSON 格式）
  - 抽卡详情（JSON 格式）
  - **AI 原始回复**（未经解析的完整文本，便于排查分节解析问题）

### 3.7 安全防护

#### 3.7.1 XSS 注入防护

为防止 XSS 注入攻击，所有用户输入和 API 返回的 HTML/Markdown 内容必须经过 DOMPurify 净化处理。

**DOMPurify 配置**：
```javascript
import DOMPurify from 'dompurify';

const md = new markdownit({...});

// 净化 HTML 输出
const clean = DOMPurify.sanitize(dirty);
```

**需要净化的场景**：
| 场景 | 来源 | 净化方式 |
|------|------|----------|
| AI 解读内容 | MiniMax API 返回 | markdown-it 解析后 DOMPurify 净化 |
| 用户问题输入 | 用户手动输入 | DOMPurify.sanitize() |
| 占卜结果数据 | 牌阵数据 | 根据数据类型选择性净化 |

**安全规则**：
- 禁止将用户输入直接插入 HTML
- 所有动态内容必须经过净化处理
- 启用 `ALLOWED_TAGS` 和 `ALLOWED_ATTR` 白名单

### 3.8 访问记录与统计

#### 3.7.1 功能概述

在"我的"页面新增"访问统计" Tab，展示用户的占卜访问记录和统计数据。

**设计原则**：
- 隐私优先：不记录具体提问内容
- 数据最小化：仅收集必要的统计信息
- 本地存储：数据存储在 localStorage，支持清除
- 实时更新：设备统计每小时重新计算并存储

#### 3.7.2 记录数据

**访问记录项 (VisitRecord)**：
```javascript
{
  sessionId: String,      // 会话ID（UUID）
  anonymizedIp: String,   // 脱敏IP（如 "202.199.*.*"）
  questionCount: Number,  // 该会话提问次数
  firstVisit: String,     // 首次访问时间 (ISO 8601)
  lastVisit: String,      // 最后访问时间 (ISO 8601)
  deviceType: String,     // 'desktop' | 'tablet' | 'mobile'
  osType: String,         // 'Windows' | 'macOS' | 'Linux' | 'Android' | 'iOS' | 'Other'
  browser: String         // 'Chrome' | 'Firefox' | 'Safari' | 'Edge' | 'Other'
}
```

**脱敏规则**：
- IPv4：保留前两段，后两段替换为 `*`（如 `202.199.*.*`）
- OS 类型从 User-Agent 中提取，不存储完整字符串
- 浏览器类型从 User-Agent 中提取，不存储完整字符串
- 会话ID使用 UUID v4，不关联真实身份

- ❌ 完整 IP 地址
- ❌ 完整 User-Agent

#### 3.7.4 存储方案

**localStorage Key**: `tarotqa_visit_stats`

**数据结构**：
```javascript
{
  version: "1.1",           // 数据版本号，用于迁移
  records: [VisitRecord],   // 访问记录数组
  stats: {
    totalSessions: Number,
    totalQuestions: Number,
    deviceStats: {
      desktop: Number,      // 桌面会话数
      tablet: Number,        // 平板会话数
      mobile: Number         // 手机会话数
    },
    osStats: {
      Windows: Number,
      macOS: Number,
      Linux: Number,
      Android: Number,
      iOS: Number,
      Other: Number
    },
    lastUpdated: String,     // ISO 8601
    lastDeviceStatsUpdate: String  // 上次设备统计更新时间
  }
}
```

**更新频率**：
- `stats.deviceStats` 和 `stats.osStats` 每小时更新一次
- 存储 `lastDeviceStatsUpdate` 时间戳，避免频繁重算
- 页面加载时检查是否超过1小时，超过则重新计算并更新

**存储限制**：
- 最多保留 100 条会话记录
- 超过时自动清除最早的记录
- 支持一键清除所有数据

#### 3.7.5 记录触发时机

**创建新会话**：
- 首次访问网站时（localStorage 无记录）
- 会生成新 sessionId 并记录首次访问时间

**更新会话记录**：
- 用户点击"AI深度解读"按钮时，questionCount +1
- 每次页面加载时更新 lastVisit、deviceType、osType 信息

**更新设备统计**：
- 页面加载时检查距离上次更新是否超过1小时
- 超过1小时则重新遍历 records 计算 deviceStats 和 osStats

## 9. 版本规范

### 9.1 版本号格式

采用语义化版本 (SemVer) 格式：`主版本.次版本.修订号`

- **主版本 (Major)**：不兼容的重大功能变更
- **次版本 (Minor)**：向后兼容的新功能添加
- **修订号 (Patch)**：向后兼容的问题修复和小优化

**版本示例**：v2.9.1

### 9.2 版本更新规则

| 变更类型 | 更新示例 | 说明 |
|----------|----------|------|
| 新功能 | v2.9.0 → v2.10.0 | 添加新功能 |
| 功能优化 | v2.9.0 → v2.9.1 | 优化现有功能 |
| Bug 修复 | v2.9.0 → v2.9.1 | 修复问题 |
| 重构 | v2.9.0 → v3.0.0 | 破坏性变更 |

**当前版本**：v2.9
